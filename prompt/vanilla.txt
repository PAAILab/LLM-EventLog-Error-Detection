Error Types to Detect

Instructions for examples
In the following Examples, each event is represented using the structure:
``` case_id, activity, timestamp, resource, error_annotation ```
- If the error_annotation field is empty, the event is clean and the recorded activity label is assumed to be correct.
- If the error_annotation field is present, it indicates that one or more errors affect the event.
- The error type(s) specify the nature of the corruption.
- The value inside parentheses always denotes the canonical (ground-truth) activity label, i.e., the label that should have been recorded without errors.
- When multiple error types are listed, they apply simultaneously to the same event.

1. Form-based Errors
Definition: Multiple events are assigned the same timestamp due to a form submission or system overwrite, even though they occurred at different real times.

2. Polluted Errors
Definition:  Activities contain machine-generated suffixes or prefixes appended to valid activity labels, introduced by automated systems or logging tools.

3. Distorted Errors
Definition: Activities contain typos, character swaps, or spelling errors in the base label text.

4. Synonymous Errors
Definition: Two or more distinct activity label strings exist in the log that refer to the same canonical business action. Synonymy is detectable from the label text alone — the labels are near-paraphrases of each other (e.g. "assess application" and "review application" are both present in the log and mean the same thing).

5. Collateral Errors
Definition: Duplicate events caused by logging artifacts, where the same activity instance is recorded multiple times without advancing the case state. This includes exact duplicates (same case, activity, and timestamp) and near-duplicates that occur within an implausibly short interval and share identical contextual attributes.

6. Homonymous Errors
Definition: The same activity label is used to represent different underlying business actions with different meanings. Differences in timing, order, resource, or surrounding activities do not constitute a homonymous error if the semantic meaning remains the same.

Multiple Errors
CRITICAL: A single row can have multiple error types simultaneously. Error types operate on independent layers:
Temporal structure: form-based (timestamp collisions)
Mechanical suffix: polluted (machine-generated suffixes)
Base-text quality: distorted (typos, character corruption)
Semantic meaning: synonymous (different wording, same meaning)
Duplication: collateral (logging artifacts)
Identifier noise: homonymous (same label, different meanings)
